História dos Computadores ?

Os primeiros computadores ocupavam salas inteiras, computadores vinham com ideias de grandes maquinas de cálculos, para contas complexas, também vinham com interesse em criptoanálises, para descobrir as criptografias durante as guerras e descobrirem mensagem que eram enviadas pelos ‘inimigos’. Sendo eles ENIAC do lado dos EUA e Colossus na Alemanhã.

Curiosidade:
Bug foi um termo que surgiu nessas primeiras gerações de computadores, onde os computadores eram maiores é mais "Frágeis". Bug significa inseto, nisso quando insetos como moscas, mosquitos… pousavam ou entravam nestes grande computadores, acabam danificando os equipamentos, nisso sempre que dava erro, falavam ‘deu bug’.

As próximas gerações, foram evoluindo constantemente em questões tecnológicas, grandes CPUs foram substituídas por pequenas maquinas, fazendo os computadores da época serem mais portáteis, e tornando mais eficazes. Equipamentos da primeira geração até hoje em dia passaram por um processo de evolução tecnológica, de Grande processadores para Microprocessadores, de Chips, Transitors, Capacitores etc… A evolução chegando até os dias de hoje. Surgindo grandes nomes nesse processo desde os antigos para os Charles Babbage, Alan Turing para os novos Bill Gates, Steve Jobs, que inovaram nesta parte do mercado, para a técnologia e informação do mundo.

História da Internet ?

A internet começou como uma rede experimental chamada ARPANET nos anos 1960. Na década de 1980, o protocolo TCP/IP foi adotado como padrão. A popularização da internet ocorreu nos anos 1990, com o surgimento da World Wide Web. A banda larga e dispositivos conectados impulsionaram seu crescimento nos anos 2000. A internet transformou a comunicação, o comércio, as redes sociais e se tornou uma ferramenta essencial em nossas vidas. O que são Clientes ?

Quando falamos de Web diferente do uso que nós fazemos normalmente, o termo "cliente" se refere ao dispositivo ou programa que usamos para acessar a internet e interagir com os serviços online. Um cliente é responsável por solicitar e receber informações de um servidor, ou seja ao meu ver, ele é o consumidor das informações entregadas pelo ‘Servidor’.

Por exemplo, um navegador da web, como o Google Chrome ou o Mozilla Firefox, é um tipo de cliente. Ele permite que você acesse sites, faça pesquisas e visualize conteúdo online. O navegador envia solicitações para os servidores que hospedam esses sites e recebe as respostas correspondentes, exibindo-as para você.

- Resumidamente cliente é a parte da interação na Web que você utiliza para acessar e interagir com os serviços online, enquanto o servidor é responsável por fornecer as informações e responder às solicitações feitas pelo cliente.

O que são Servers ?

O que foi explicado sobre o outro lado/contexto, o "servidor" é um computador ou sistema que armazena e fornece os recursos e informações solicitados pelos clientes. Os servidores são responsáveis por receber as solicitações dos clientes, processá-las e enviar as respostas apropriadas de volta aos clientes.

Os servidores são configurados para hospedar sites, aplicativos, serviços de e-mail, bancos de dados e outros recursos online. Eles são responsáveis por armazenar os arquivos necessários para exibir os sites, processar as informações enviadas pelos clientes e executar as funções solicitadas.

Quando um cliente faz uma solicitação para acessar um site, por exemplo, o servidor recebe essa solicitação e envia os arquivos HTML, CSS, JavaScript, imagens e outros recursos necessários para exibir o site no navegador do cliente. O servidor também pode processar formulários enviados pelo cliente, executar tarefas específicas e interagir com bancos de dados para recuperar ou armazenar informações. Só para isso á rede tem que ser mais rápida e eficaz.

Em resumo, o servidor é responsável por fornecer os recursos e informações solicitados pelos clientes. Ele processa as solicitações, executa as funções necessárias e envia as respostas de volta aos clientes. É o lado da interação na Web que armazena e disponibiliza os dados e serviços que os clientes acessam.

Como é uma estrutura básica de um HTML ?

HTML é a linguagem de marcação utilizada para estruturar e formatar o conteúdo de uma página da web. A estrutura básica do HTML consiste em várias tags que são usadas para definir diferentes elementos e seções da página.

Vamos começar com as tags do <h1> ao <h6>:

  • <h1> ao <h6>:Essas tags são usadas para definir os títulos ou cabeçalhos da página, sendo <h1> á maior e <h6> á menor.
  • <p>: Essa tag é usada para definir parágrafos de texto.
  • <mark>: Essa tag é usada para destacar um trecho de texto, geralmente aplicando uma cor de fundo.
  • <small> Essa tag é usada para definir um texto menor, geralmente utilizado para notas de rodapé ou informações secundárias.
  • <i> Essa tag é usada para definir texto em itálico.
  • <u> Essa tag é usada para sublinhar um trecho de texto.
  • <strong> Essa tag é usada para enfatizar um trecho de texto, geralmente exibindo-o em negrito.
  • <ol> Essa tag é usada para criar uma lista ordenada, onde cada item é numerado.
  • <ul> Essa tag é usada para criar uma lista não ordenada, onde cada item é marcado com um símbolo, como um ponto ou um traço.
  • <li>: Essa tag é usada para definir um item em uma lista.
  • <a>: Essa tag é usada para criar um link, permitindo que os usuários cliquem para navegar para outra página ou recurso.
  • <hr>: Essa tag é usada para inserir uma linha horizontal na página, geralmente para separar seções.
  • <sub>: Essa tag é usada para definir texto como sobrescrito, exibindo-o abaixo da linha base.
  • <sup>: Essa tag é usada para definir texto como sobrescrito, exibindo-o acima da linha base.
  • <blockquote>: Essa tag é usada para criar um bloco de citação, geralmente com recuo e estilo diferenciado.
  • <font>: Essa tag é usada para definir o estilo de fonte do texto. No entanto, é considerada obsoleta e o uso de CSS é recomendado para estilizar o texto.
  • <del>: Essa tag é usada para indicar que um trecho de texto foi deletado ou removido.
  • <abbr>:Essa tag é usada para definir uma abreviação ou acrônimo, permitindo que o usuário veja a versão completa ao passar o mouse sobre o texto.

Entendendo a diferença entre listas ordenadas e não ordenadas.

No HTML temos duas formações diferentes para criar listas, sendo elas as listas ordenadas e não ordenadas, onde devemos avaliar bem na hora de utiliza-las, pois ao mesmo tempo que facilitam muito processos de organização, quando confudidas podem afetar diretamente quando alguém for for utiliza-las:


Lista Ordenada (' <ol> '):

Essa tag usada para criar uma lista ordenada, na qual cada item é numerado sequencialmente. Os itens em uma lista ordenada são exibidos com números, letras ou símbolos, dependendo do tipo de numeração especificado. Por padrão, os navegadores geralmente usam números para numerar os itens. A ordem em que os itens são listados irá seguir uma sequência específica definida pelo autor.

Segue abaixo um exemplo de uma lista ordenada:

  1. Item 1
  2. Item 2
  3. Item 3
Então de prefêrencia, sempre que for usa-las avalie o contexto, os itens que vão ser incluidos, o grau de importância de cada item, se possuem uma ordem específicas.

Lista Não Ordenada (' <ul> '):

Essa tag usada para criar uma lista não ordenada, na qual cada item é marcado com um símbolo, geralmente um ponto ou um traço. Os itens em uma lista não ordenada não seguem uma sequência específica e a ordem não tem necessáriamente um significado, no caso geralmente é indiferente a ordem.

Segue abaixo um exemplo de uma lista não ordenada:
  • Item A
  • Item B
  • Item C
Sempre que for usa-las avalie o contexto, os itens que vão ser incluidos para não colocarem algo de etapas em uma ordem aleátoria :)

Sobre Links.

Sobre links, temos diversas possibilidade, tanto que conseguirmos ancorar na própria aba, quanto ir a outra páginas, puxar imagens de algum site ou alguma fonte... Temos diversas possibilidades diferentes para estar utilizando linkagem, e facilitando o processo para o usuário e assim melhorando sua experiência em nossa página.



Ancoragem em outra página:

Segue abaixo uma explicação abaixo de uma âncora para outra página totalmente diferente:

Utilizando o <a href="site.com">Visite o Site.com<a/> conseguimos levar o usuário em outra página, que muitas vezes não terá conexão com a página atual.



Ancoragem na mesma página:

Segue abaixo uma explicação abaixo de uma âncora para a mesma página:

Você também pode criar âncoras em uma página, permitindo que os usuários pulem para seções específicas do conteúdo na mesma página. Para fazer isso, você precisa adicionar um id à tag do elemento de destino e usar esse id como valor do atributo href no link. Aqui está um exemplo:

<a href="#secao1">Vá para a Seção 1<a/>

<h2 id="#secao1">Seção 1<a/>
